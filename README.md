# zk-Trace for agents
This is a prototype implementing an IVC-based Zero-Knowledge prooving procedure for activity traces generated by a multi-agent system.

## Foundation
This implementation uses Microsoft's Nova project as the backend for the proof generation. Nova is the first work to introduce recursive zk-proofs based on Incrementally Verifiable Computation (IVC), allowing one to prove statements about a history of states, without the need to keep track of the past states. 
The advantage of recursive folding is the near constant size of the running accumulated instance, for which a proof can be generated at any given point.
Some property about a history of actions, such as performed tool calls by the agent, can thus be captured by a fixed constraint circuit unstead of a composition of several circuits.

Recursive proofs are widely used within zero-knowledge ML verification. However the technique is still predominantly used in heavy weight procedures, typically for prooving the inference process of a network-model architecture.
This project focuses instead solely on trace-friendly artifacts, such as tool calls, which are much simpler and computationally less intensive in comparison.

The aim of this project is to
- show the feasibility of realtime zk-prooving for agentic systems, limited to trace verification
- design policies about trace sequences, which are then to be translated into corresponding constraint step-circuits

## Implementation
To run the project you need to install the Nova backend locally. Follow these instructions from the Nova repository:
https://github.com/microsoft/Nova.git

Further, you also need an API key of an LLM provider of your choice (for ex. OpenAI or Claude) to use for the agents. secrets.py is omitted here. Please create the file locally in your enviuronment and paste the secret API key into it.

With Nova installed, you can run the following examples for testing recursive proof accumultation:
- agent_plcy_test.rs
- agent_policy_test2.rs
- agent_policy_test3.rs

In order to test a streaming real-time proof aggregation, furst run agent_policy_stream3.rs which will open a pipe and listen for incomming inputs. Then run multi_agent_main.py. 
The running agent from multi_agent_main.py will occasionally generate a number and write it into the pipe. The .rs policy script will detect the incomming input and generate a corresponding step circuit for it. The process can be observed at runtime in the console windows for the agent and the window for the recursive proof process.
