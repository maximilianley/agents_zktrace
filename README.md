# zk-Trace for Agents

A prototype implementing an IVC-based zero-knowledge proving procedure for activity traces generated by a multi-agent system.

---

## Foundation

This implementation uses Microsoft's **[Nova](https://github.com/microsoft/Nova)** project as the backend for proof generation.

Nova is the first work to introduce recursive zk-proofs based on **Incrementally Verifiable Computation (IVC)**, allowing one to prove statements about a history of states without the need to keep track of past states.

The advantage of recursive folding is the near-constant size of the accumulated running instance, for which a proof can be generated at any given point.

Properties about a history of actions—such as tool calls performed by an agent—can therefore be captured by a fixed constraint circuit rather than a composition of multiple circuits.

Recursive proofs are widely used within zero-knowledge ML verification. However, the technique is still predominantly applied to heavyweight procedures, typically proving the inference process of neural-network architectures.

This project instead focuses solely on **trace-friendly artifacts**, such as tool calls, which are significantly simpler and computationally less intensive.

### Goals

- Demonstrate the feasibility of real-time zk-proving for agentic systems, limited to trace verification.
- Design policies over trace sequences and translate them into corresponding constraint step circuits.

## Implementation

### Prerequisites

1. Install the Nova backend locally by following the instructions in the official repository:
   - **Nova:** https://github.com/microsoft/Nova

2. Obtain an API key from an LLM provider of your choice (e.g. OpenAI or Claude).

3. Create a local `secrets.py` file and place your API key inside it.

> Note: `secrets.py` is intentionally omitted from the repository.

## Real-Time Streaming Proof Aggregation

### Step 1

Start the proof process:

```bash
cargo run --bin agent_policy_stream3
```

This opens a pipe and listens for incoming inputs.

### Step 2

Run the multi-agent application:

```bash
python multi_agent_main.py
```

### What Happens

- The agent occasionally generates a number and writes it into the pipe.
- `agent_policy_stream3.rs` detects the incoming input.
- A corresponding step circuit is generated.
- The recursive proof state is updated through folding.

You can observe the process in real time through:

- The agent console (`multi_agent_main.py`)
- The recursive proof process console (`agent_policy_stream3.rs`)
